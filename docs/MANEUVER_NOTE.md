# Why I Built This for Maneuver

Maneuver's public work appears focused on practical applied AI: workflows, Voice AI, AI agents, operational dashboards, integrations, and measurable business outcomes.

The hospitality case study stood out because it is not just an AI concierge problem. It is a reliability problem.

Guest communication involves incomplete data, angry users, policy constraints, refund pressure, safety issues, escalation, and channel switching.

So I built GuestOps Chaos Lab to explore the layer I would want before deploying an AI concierge to real guests:

- Can the agent avoid hallucinating?
- Can it use tools correctly?
- Can it escalate when needed?
- Can it avoid unauthorized promises?
- Can it admit missing data?
- Can the team inspect why it failed?

This project is a small proof-of-work showing how I think: product-first, eval-aware, backend-aware, and focused on applied AI that survives real workflows.
