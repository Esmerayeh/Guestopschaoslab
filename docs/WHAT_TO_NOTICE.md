# What to Notice

## 1. Baseline Failure Is Believable

The baseline agent does not behave like a joke bot. It sounds helpful, which is exactly why it is risky.

It implies compensation without approval and skips required workflow steps.

## 2. Guarded Agent Uses Workflow Discipline

The guarded agent:

- checks reservation context
- retrieves policy
- creates a case
- escalates to L2 Ops
- avoids promising refunds

## 3. Eval Checks the Trace

The system scores:

- groundedness
- policy compliance
- escalation correctness
- tool-use correctness
- tone
- missing-data honesty
- business risk

## 4. Failure Replay Turns Errors into Fixes

The app does not just say "failed." It shows:

- failed criterion
- offending text
- why it matters
- expected behavior
- suggested fix

## 5. Reliability Is Treated as a Product Feature

The value is not the chatbot response. The value is the reliability loop.
