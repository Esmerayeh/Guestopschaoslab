# UI / UX Brief

## Direction

GuestOps Chaos Lab should feel like a premium AI operations cockpit: dark, precise, serious, and practical.

The design should be Maneuver-aligned in spirit: useful over flashy, workflow-first, and visibly concerned with production outcomes.

## Visual Tokens

- background: `#05070D`
- panel: `#07111F`
- panel secondary: `#0B1628`
- primary blue: `#1D6BFF`
- cyan: `#7DD3FC`
- text: `#F8FAFC`
- muted: `#94A3B8`
- success: `#22C55E`
- warning: `#F59E0B`
- danger: `#EF4444`

## Typography

- Primary UI font: Inter / Geist / system sans.
- Trace and metric font: IBM Plex Mono-style fallback through `ui-monospace`.
- Use compact dashboard headings, not oversized marketing blocks.

## Core Screens

- Homepage: proof-of-work framing, reviewer path, architecture strip, KPI cards.
- Reviewer Demo: curated Refund Trap flow.
- Scenario Gallery: risk cards and compare CTAs.
- Run Lab: guest simulator, agent response, structured decision, eval scorecard, trace timeline.
- Agent Comparison: baseline vs guarded score/risk delta.
- Failure Replay: offending line, risk, expected behavior, guarded fix.
- Eval Library: scoring dimensions.

## UX Principles

- Make reliability visible.
- Prefer traces and evidence over claims.
- Keep microcopy practical and confident.
- Use controlled personality sparingly.
- Avoid toy-like colors, oversized fluff, or unsupported production claims.
